 Perform precise OCR extraction from this academic grading ledger sheet.
                    IMPORTANT TABLE STRUCTURE INFORMATION:
                    This ledger contains multiple subject blocks arranged horizontally in the same student row.
                    The header pattern is repeated:
                    SYMBOL | REG. NO. | NAME OF THE STUDENT | SUBJ | TH | SUBJ | TH | PR | TOT | SUBJ | TH | PR | TOT | SUBJ | TH | PR | TOT | SUBJ | TH | PR | TOT | TOTAL | REMARKS

                    SYMBOL | REG. NO. | NAME OF THE STUDENT | SUBJ | TH | SUBJ | TH | PR | TOT | SUBJ | TH | PR | TOT | SUBJ | TH | PR | TOT | SUBJ | TH | PR | TOT | TOTAL | REMARKS
                    SYMBOL | REG. NO. | NAME OF THE STUDENT | SUBJ | TH | SUBJ | TH | PR | TOT | SUBJ | TH | PR | TOT | SUBJ | TH | PR | TOT | SUBJ | TH | PR | TOT | TOTAL | REMARKS
                                                DOB                     | SUBJ | TH | PR | TOT
                    Each SUBJ column starts a new subject block. Do NOT treat all TH/PR/TOT columns as belonging to the same subject.
                    COLUMN ALIGNMENT RULES:
                    1. Read each student row from left to right.
                    2. Student information columns:
                    - SYMBOL
                    - REG. NO.
                    - NAME OF THE STUDENT
                    - DOB
                    3. After student information, extract each subject block separately.
                    Example:
                    C.ENG | A | C.NEP | A | PHY | A | 19 | 19 | CHEM | A | 15 | 15 | BIO | A | 15 | 15
                    Must become:
                    [
                    {
                    "subject_name": "C.ENG",
                    "theory": "A",
                    "practical": null,
                    "total": null
                    },
                    {
                    "subject_name": "C.NEP",
                    "theory": "A",
                    "practical": null,
                    "total": null
                    },
                    {
                    "subject_name": "PHY",
                    "theory": "A",
                    "practical": "19",
                    "total": "19"
                    },
                    {
                    "subject_name": "CHEM",
                    "theory": "A",
                    "practical": "15",
                    "total": "15"
                    },
                    {
                    "subject_name": "BIO",
                    "theory": "A",
                    "practical": "15",
                    "total": "15"
                    }
                    ]

                    VERY IMPORTANT:
                    - C.ENG and C.NEP generally contain only TH marks.
                    - Do NOT assign later subject PR/TOT values to C.ENG or C.NEP.
                    - If a subject block has no PR or TOT column value, return null.
                    - Never move a value horizontally between subject blocks.
                    MARK EXTRACTION RULE:
                    Extract marks exactly as printed.
                    Preserve:
                    - letters (A, B, etc.)
                    - numbers
                    - asterisks (*)
                    - handwritten corrections
                    - special symbols
                    Do not calculate totals.
                    Do not infer missing marks.
                    TOTAL COLUMN:
                    The final TOTAL column at the far right is the student's grand total.
                    Rules:
                    - Extract only if a number is visibly printed.
                    - If blank, return null.
                    - Never replace blank with 0.
                    Some students have additional subject rows below the main row.
